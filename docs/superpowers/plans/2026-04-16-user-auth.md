# User Auth & Accounts Implementation Plan

## 1. Goal
Implement a robust authentication system using **NextAuth.js** (Auth.js) with Google and Email login providers.
Once authenticated, we need to map their `userCards` (Wallet) and tracked flights/alerts to a permanent `User` record in our Prisma SQLite database, allowing them to access their profile across multiple devices.

## 2. Components
1. **NextAuth Setup**:
   - Install `next-auth` and `@auth/prisma-adapter`.
   - Update `prisma/schema.prisma` to include the standard Auth.js models (`User`, `Account`, `Session`, `VerificationToken`).
   - Configure `src/app/api/auth/[...nextauth]/route.ts`.

2. **Database Synchronization**:
   - Add a `savedCards` JSON/String field to the `User` model.
   - When a user logs in, we hydrate the `user-store` (Zustand) with their `savedCards` from the database.
   - When a user updates their wallet via the `WalletModal`, we sync the changes back to the database via a Server Action.

3. **UI Updates**:
   - Add a User Dropdown / Login button to the global navigation bar (`src/components/layout/Header.tsx` or similar).
   - Display their Avatar and Name if logged in.
   - Add a "My Profile" page where they can manage their account settings and view past searches.

## 3. Execution Steps
1. Install dependencies: `npm install next-auth @auth/prisma-adapter`.
2. Update `schema.prisma` and run `npx prisma db push`.
3. Create `src/lib/auth.ts` (Auth options) and the NextAuth route handler.
4. Update the global navigation to include "Sign In" / User Menu.
5. Create a Server Action (`syncWallet`) to update the user's cards in the DB when they save changes in the `WalletModal`.
